# /generate-content

X投稿向けコンテンツを生成するコマンド。voice-guideに従って、5件のネタを生成します。

## 準備（手動）

1. **NotionのソースDBを開く**
   - 直近1週間のアイデア・実験ログ・記事まとめを確認
   - 「X投稿ネタになりそう」な項目を3〜5件ピックアップ
   - **コピー**：ページのURL と 概要をコピー

2. **フィードバックDBを確認（初回は省略可）**
   - 過去1週間で反応が良かった投稿を3〜5件ピックアップ
   - タイトル・本文・反応数をコピー
   - 「何が反応良かったか」を1〜2行で分析

## 実行

```
/generate-content
```

プロンプトに以下を入力：

```
【voice-guide】
[.claude/voice-guide.md の全内容をコピペ]

【本日のソースデータ】
[ステップ1で準備したソースDB内容をペースト]

【過去のフィードバック】
[ステップ2で確認した反応が良かった投稿パターンをペースト]
（初回は「分析データなし。voice-guideに従い、実験的・カジュアルなトーンで生成してください」でOK）

【実行指示】
上記のvoice-guideを厳密に守って、今日のX投稿向けネタを5件生成してください。
出力フォーマットは「4. 出力フォーマット」に従ってください。
```

## 出力内容

5件のネタが以下のフォーマットで出力されます：

```
【ネタ No.1】
📌 タイトル案（X用）：
[140字以内]

📝 本文ドラフト（X用）：
[実投稿可能な長さ]

🔗 ブログ展開版：
[深い解説が必要な場合のみ]

💡 想定反応と理由：
[なぜ刺さるのか]

⚙️ 必要な追加アクション：
[画像生成、コード例など]

📌 使用するソース：
[ソースDBのURL]

【ネタ No.2】
[以下同様...]
```

## 出力後

### 1. Output DBに登録（手動）
- 生成結果をコピー
- NotionのOutput DBを開く
- 各ネタごとに新規ページ作成
- 内容を貼り付け
- ステータスを「下書き」に設定

### 2. X投稿（手動）
- Output DBから下書きを選択
- 本文ドラフトをコピー
- X（Twitter）に投稿
- 投稿URLをOutput DBに記載

### 3. 反応記録（24h後）
- インプレッション・いいね・リプ数を確認
- フィードバックDBに記録
- 「何が反応良かったか」を1〜2行で分析メモ

## よくある質問

**Q. 生成に何分かかる？**
→ 5分〜10分。ソースデータの量による。

**Q. 毎日実行すべき？**
→ 推奨は毎朝1回。ソースDBに3〜5件データが溜まったら実行のが品質良い。

**Q. ネタが気に入らなかったら？**
→ voice-guideの「トーン・表現ガイドライン」を調整するか、ソースデータを追加して再生成。

**Q. Output DBへの自動登録はできる？**
→ 現在は手動。将来的にNotion APIスキルで自動化予定。

## トラブルシューティング

**「ネタが何か薄い」**
→ ソースデータを追加してから再生成。「3件 → 5件」に増やすと質が上がることが多い。

**「トーンが固い」**
→ voice-guide の「具体例」セクションを見直し。フィードバックから「どのネタが反応良かったか」を次回プロンプトに含める。

**「同じテーマばかり」**
→ ソースDBに多様なテーマ（実装ログ・記事まとめ・トレンド・失敗ネタなど）を意識的に追加する。
