# Notion統合ルール

## 読み取り操作（検索・参照）

- いつでも自由に知識ベースから検索・参照できる
- `/notion-search` でキーワード検索
- `/notion-read-page` でページ全体を取得
- 実装前に関連する仕様・設計ドキュメントを必ず確認する

## 書き込み操作（作成・更新）

### 作成時
- 新規ページ作成前に目的・内容を確認する
- テンプレートがあれば使う（例：「実装ログ」テンプレート）
- タイトル・タグ・メタデータは統一フォーマットで

### 更新時
- 既存ページへの更新は慎重に
- 破壊的な変更（削除）は実行前に確認を取る
- 更新内容は明確なコミットメッセージで記録

## データベース管理

- 知識ベース（読み取り専用推奨）: 仕様・設計ドキュメント
- 実装ログ（追記可能）: 実装完了後に新規ページを追加
- タスク管理（将来拡張予定）: 現在は参照のみ

## 認証情報

- Notion Integration Token は `CLAUDE.local.md` で管理
- `.mcp.json` は git管理外
- 本番データベースと開発環境を明確に分離する

## トラブルシューティング

- 接続エラー: MCPサーバーの接続確認（`claude mcp list`）
- 権限エラー: Notion統合の権限設定を確認
- 404エラー: ページID・データベースIDが正確か確認
